Универсальные списки


Универсальные списки – структура данных позволяющая задать различные параметры для отдельных товаров и прайс листов.
Основные типы списков 
Разделы и типы списков, а так же параметры для списков могут создаваться пользователями самостоятельно. Это позволяет управлять ценообразованием максимально гибко. 
Раздел Ценообразование
•	Максимальная наценка – применяется для ограничения наценки сверху. Чаще всего попадают препараты, которые необходимо распродать 
•	Максимальная цена – списки контролирующие максимальную цену. Примеры применения: ограничение максимальной цены производителем
•	Минимальная наценка – используется для указания критически наценок. 
•	Минимальная цена – используется для ограничения нижней границы цены. Пример применения: товар с ожидаемым подорожанием, дефициты, эксклюзивы и т.д.
•	Фиксированная цена – чаще всего применяется для товаров с особыми условиями от потсавщика
•	ПДЦ по ЖВ – предельно допустимые цены (регулируется государством(
•	Критички – тоже что и минимальная наценка

При создании\редактировании списка откроется окно
В верхней части окна заполняется шапка списка. 
Поля:
•	Код – присваивается автоматически не редактируется пользователем
•	Наименование – произвольная строка названия списка. Желательно именовать списки таким образом, что бы в последствии было однозначно понятно для чего он был использован. 
•	Статус – Активен\не активен. В работе алгоритмов применяются только активные списки. Для отключения списка можно просто перевести его в соответствующий статус
•	Раздел и тип – выбираются пользователем из выпадающего списка. Используются алгоритмами, если необходимо обработать несколько списков сразу.
•	Начало и окончание действия – определяют период действия списков. Если период истек или не наступил, то данный список не будет применяться в алгоритмах ЦО. 
Левая табличная часть списков содержит препараты и параметры для них. Препараты можно добавлять по одному, а так же можно импортировать из внешнего файла. Для имопрт нажать кнопку

1) Вывод по проекту
Проект по сути — это движок ценообразования для аптечных прайс-листов.
То есть система должна:
•	взять товары (SKU), 
•	взять себестоимость / базовую цену, 
•	взять минимальные цены конкурентов (МЦК), 
•	взять настройки ценового формата (ЦФ), 
•	применить правило ценообразования, 
•	учесть универсальные списки, 
•	выдать готовый прайс-лист, 
•	и показать объяснение, почему цена получилась именно такой. 
То есть это не просто “табличка с ценами”, а правиловая система принятия решений.

4) Критические вопросы к бизнесу / заказчику
A. Что является базой для расчета цены?
Нужно четко определить:
От чего стартуем?

•	от себестоимости 
•	от действующей нашей цены 
•	от рекомендуемой наценки 
•	от МЦК 
•	от закупочной цены 
•	от цены поставщика 
Нужно зафиксировать:
Базовая цена для расчета = cost_price (или другая сущность)

B. Формула “Под цены конкурентов”
Это главный алгоритм, и его надо формализовать.
Сейчас из описания логика выглядит так:
Цена = цена конкурента − прогиб,
но не ниже критической наценки,
и, возможно, не выше рекомендованной / максимальной / фиксированной границы.
Но неясно, в каком именно порядке это применяется.
Нужно зафиксировать:
Например так:
1.	Берем МЦК 
2.	Применяем прогиб 
3.	Проверяем минимальную наценку 
4.	Проверяем максимальную наценку 
5.	Проверяем минимальную цену 
6.	Проверяем максимальную цену 
7.	Проверяем фиксированную цену 
8.	Применяем округление 
Или другой порядок — но он должен быть официально утвержден.
________________________________________

•	МЦК даёт цену 420 
•	минимальная цена по списку = 450 
•	фиксированная цена = 470 
•	максимальная цена = 430 
Какая цена должна победить?
Это нужно определить как таблицу приоритетов.

•	брать только по региону 
•	брать только по активным ПЛК 
•	учитывать коэффициент поставщика? 
Это должно быть четко описано.

•	товар уходит в “непересекаемый сегмент” 
•	цена не формируется вообще 
•	применяется резервный алгоритм

7. Алгоритм расчета цены (MVP)
Для каждого SKU выполняется следующий порядок:
________________________________________
Шаг 1. Определить МЦК
Для SKU выбирается минимальная цена конкурента среди всех активных привязанных ПЛК данного ЦФ.
Формула:
MCC = min(competitor_price)
Если конкурентных цен нет:
MCC = null
________________________________________
Шаг 2. Определить стартовую цену
Если MCC != null:
base_price = MCC * (1 - bend_percent / 100)
где:
•	bend_percent = прогиб для данного ЦФ / SKU / ценового веса 
Если MCC == null:
base_price = cost_price * (1 + recommended_margin / 100)
________________________________________
Шаг 3. Проверить критическую наценку / минимальную наценку
Минимально допустимая цена по наценке:
min_allowed_by_margin = cost_price * (1 + min_margin / 100)
Если:
base_price < min_allowed_by_margin
то:
base_price = min_allowed_by_margin
________________________________________
Шаг 4. Проверить максимальную наценку (если задана)
Максимально допустимая цена:
max_allowed_by_margin = cost_price * (1 + max_margin / 100)
Если:
base_price > max_allowed_by_margin
то:
base_price = max_allowed_by_margin
________________________________________
Шаг 5. Проверить минимальную цену из универсальных списков
Если существует активное ограничение:
base_price < min_price
то:
base_price = min_price
________________________________________
Шаг 6. Проверить максимальную цену из универсальных списков
Если существует активное ограничение:
base_price > max_price
то:
base_price = max_price
________________________________________
Шаг 7. Проверить фиксированную цену
Если по SKU есть активная фиксированная цена:
base_price = fixed_price
Фиксированная цена имеет наивысший приоритет
(если бизнес не утвердит другой порядок)
________________________________________
Шаг 8. Применить округление
Пример MVP:
price = round(base_price, 0)
или до 1 тенге / 5 тенге / 10 тенге — в зависимости от настройки.
________________________________________

1.	Фиксированная цена 
2.	Минимальная цена 
3.	Максимальная цена 
4.	Минимальная наценка 
5.	Максимальная наценка 
6.	Прогиб под МЦК

Настройки
Основная закладка, позволяющая настраивать параметры конкретного ЦФ. 
Наименование и филиал являются справочной информацией. 
•	Правило ценообразование – выбор правила по которому будет формироваться цена прайса
•	Индивидуальные прогибы – если установлен флаг, то в алгоритмах правила будут использоваться соответствующие ценовые веса конкретного ценового формата. 
•	Индивидуальные диапазоны наценок – критические наценки данного ЦФ. Если не указано – будут использоваться согласно настройки алгоритма. 
•	Рек. Наценки – наценки, которые могут быть установлены в ПЛ в момент работы алгоритма «Под цены конкурента». 
Оценка сформированного ПЛ
Для просмотра результата работы. Диаграмма ПЛ, (Левое плечо- ЛП, Зона логичности, Правое плечо, не пересекаемый сегмент)
В табличной части отображается базовая информация
•	Товар
•	Цена
•	Себестоимость
•	Цена минимальная – МЦК
Отчет по ЛП в разрезе ПЛ
Левое плечо (ЛП)- кол-во sku, где сформированная цена ниже МЦК (минимальная цена конкурента)
Зона логичности (ЗЛ) - кол-во sku, где сформированная цена дороже МЦК от 0 до 3%
Правое плечо (ПП) – кол-во sku, где сформированная цена дороже МЦК более чем 3%. 
Не пересекаемый сегмент - кол-во sku, по которым нет МЦК (ошибка перекодировки или наши эксклюзивы) 
Лог формирования  цен
Показывает как менялась цена на каждом этапе работы правила ценообразования.
 Следует использовать этот функционал при настройке правил(тестирование), а так же при поступлении запросов из разряда «почему на товар была установлена такая цена»

Универсальные списки


Универсальные списки – структура данных позволяющая задать различные параметры для отдельных товаров и прайс листов.
Основные типы списков 
Разделы и типы списков, а так же параметры для списков могут создаваться пользователями самостоятельно. Это позволяет управлять ценообразованием максимально гибко. 
Раздел Ценообразование
•	Максимальная наценка – применяется для ограничения наценки сверху. Чаще всего попадают препараты, которые необходимо распродать 
•	Максимальная цена – списки контролирующие максимальную цену. Примеры применения: ограничение максимальной цены производителем
•	Минимальная наценка – используется для указания критически наценок. 
•	Минимальная цена – используется для ограничения нижней границы цены. Пример применения: товар с ожидаемым подорожанием, дефициты, эксклюзивы и т.д.
•	Фиксированная цена – чаще всего применяется для товаров с особыми условиями от потсавщика
•	ПДЦ по ЖВ – предельно допустимые цены (регулируется государством(
•	Критички – тоже что и минимальная наценка
Работа с универсальными списками
Для работы с универсальными списками нужен отдельный раздел. Где будут заводиться/храниться  списки.
При создании\редактировании списка откроется окно
В верхней части окна заполняется шапка списка. 
Поля:
•	Код – присваивается автоматически не редактируется пользователем
•	Наименование – произвольная строка названия списка. Желательно именовать списки таким образом, что бы в последствии было однозначно понятно для чего он был использован. 
•	Статус – Активен\не активен. В работе алгоритмов применяются только активные списки. Для отключения списка можно просто перевести его в соответствующий статус
•	Раздел и тип – выбираются пользователем из выпадающего списка. Используются алгоритмами, если необходимо обработать несколько списков сразу.
•	Начало и окончание действия – определяют период действия списков. Если период истек или не наступил, то данный список не будет применяться в алгоритмах ЦО. 
Левая табличная часть списков содержит препараты и параметры для них. Препараты можно добавлять по одному, а так же можно импортировать из внешнего файла. Для имопрт нажать кнопку
В окне импорта реализовать возможность вставки значений из буфера обмена. Это наиболее удобный способ заполнения списков. Просто скопируйте значения из Excel и вставьте в соответствующие колонки. 
В правой части окна со списком расположена таблица с прайс-листами на которые распространяется действие списков. Это позволяет формировать различные списки для отдельных подразделений\прайсов в рамках одного типа списков. 
Импорт и подбор прайсов работает аналогично части с товаром.

Шаг 2. Определяем базовую цену
Формула:
Цена_база = Себестоимость * (1 + Наценка)
Наценка берется из диапазона:
если цена ∈ [0–50] → 10%
если [50–100] → 8%
и т.д.

👉 ключевая формула:
Цена_конкурент = min(цены конкурентов)

Цена_наша = Цена_конкурент * (1 - Прогиб)
или (как хотят бизнес):
Цена_наша = Цена_конкурент - 2 тенге
________________________________________
Шаг 4. Ограничения (ОБЯЗАТЕЛЬНО)
1. Минимальная наценка (критичка)
Цена ≥ Себестоимость * (1 + МинНаценка)
